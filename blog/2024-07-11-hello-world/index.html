AI를 통해 사이트 구축

2025년 07월 11일

나는 백엔드 개발자다. 그래서 프론트엔드 구현법은 전혀 모른다.

이전에 개발 블로그를 운영했지만, 프론트를 전혀 몰랐기에 Jekyll을 활용해 흔히 쓰이는 템플릿을 그대로 가져다 쓰기만 했다. 그런데 그때 사용한 템플릿은 솔직히 만족스럽지 않았고, 불편해서 많이 활용하지는 못했다.

요즘은 대 AI 시대. 프론트를 잘 몰라도 AI에게 맡기면 뭐든 쉽고 빠르게 할 수 있지 않을까?

그래서 한번 만들어봤다.

1. 디자인

어떻게 만들까? 우선 블로그를 만들려면 어떤 식으로 디자인할지 정해야 했다. 회사에서 팀장님이 디자인할 때 사용하는 AI로 Readdy를 추천해주셨다.

Readdy

readdy.ai

나는 Readdy에게 간단한 기능이 포함된 블로그를 만들어 달라고 요청했다.

요청 : 개발 블로그를 만들건데 회색과 어울리는 사이트였으면 좋겠고 검색, 분류, 태그 게시글 기능이 있었으면 좋겠어 

답변: 
1. 전체 레이아웃 구조

메인 페이지는 미니멀하고 세련된 그레이스케일 컬러 스킴을 기반으로 구성됩니다. 전체적으로 #F5F5F5 배경에 #333333 텍스트를 사용하여 가독성을 확보합니다.

2. 헤더 영역

최상단에 고정된 네비게이션 바 (#FFFFFF 배경)
왼쪽에 블로그 로고/타이틀 배치
오른쪽에 검색창 배치 (회색 테두리와 돋보기 아이콘)
메뉴 항목들은 #666666 컬러로 구성
3. 카테고리/태그 섹션

헤더 바로 아래 수평 스크롤 가능한 태그 리스트 (#EEEEEE 배경)
주요 카테고리는 칩(Chip) 형태로 표시
활성화된 태그는 진한 회색(#444444) 배경으로 강조
4. 메인 콘텐츠 영역

그리드 레이아웃으로 게시글 카드 배치
각 카드는 흰색 배경에 은은한 그림자 효과
썸네일 이미지, 제목, 요약, 태그, 작성일 포함
카드 호버 시 부드러운 확대 애니메이션
5. 사이드바

오른쪽에 고정된 사이드바 (#FFFFFF 배경)
카테고리 트리 구조로 표시
인기 태그 클라우드 섹션
최근 게시물 리스트
6. 검색 기능

실시간 검색어 추천 기능
태그, 카테고리, 제목 기반 필터링 옵션
검색 결과는 관련도순/최신순 정렬 가능
7. 반응형 디자인

모바일에서는 사이드바가 햄버거 메뉴로 변환
그리드 레이아웃은 화면 크기에 따라 1~3컬럼으로 조정
검색창은 모바일에서 전체화면으로 확장
8. 기타 UI 요소

스크롤 시 최상단으로 이동하는 플로팅 버튼
페이지네이션 또는 무한 스크롤 지원
로딩 상태를 표시하는 스켈레톤 UI

간단하게 요청했을 뿐인데, 상세하게 설계하더니 다음과 같은 화면이 뚝딱 나왔다.

너무 만족스러웠다. 심지어 <code>를 누르니 아래와 같은 HTML 코드를 제공해줬다.

구현

이제 나는 이 화면 그대로 구현만 하면 됐다. 하지만 어떻게 구현해야 할지 계속 고민했다.

HTML, CSS, JavaScript가 어떤 식으로 사용되는지는 조금 알지만, 제대로 무언가를 구현해본 적은 없다. 어떻게 하면 저것과 유사하게 구현할 수 있을지 몰랐다.

그래서 Cursor를 이용해 해당 HTML 파일을 기반으로 구현해달라고 요청했다.

요청: 
@개발-블로그.html 태그한것과 같은 블로그를 만들고 싶은데 깃헙블로그로 만들거야
근데 단순히 정적인 파일로 만드는게 아니라 깃헙에서 인식해서 md 파일들을 게시글로 넣게 하고싶어 

답변:
1. Jekyll (GitHub Pages 기본 지원)
_posts/ 폴더에 YYYY-MM-DD-title.md 형식으로 md 파일을 넣으면 자동으로 블로그 글이 생성됨
커스텀 테마, Tailwind CSS, 카테고리/태그 등 확장 가능
별도 빌드/배포 과정 없이 GitHub에 푸시만 하면 자동 반영
2. Astro/Next.js/Hugo 등 최신 SSG
md/mdx 파일을 읽어 동적으로 페이지 생성
더 자유로운 커스텀, React/Vue/Svelte 등과 연동 가능
GitHub Actions 등으로 자동 빌드/배포 설정 필요

나는 이전에 Jekyll을 사용했었지만, 이번에는 다른 방법을 사용해보고 싶어서 2번 방법을 선택했다.

그러자 Cursor AI는 그중에서도 Astro를 권장하며 한 번 사용해보라고 추천했다.

그리고는 아래와 같은 답변과 함께 블로그를 만드는 과정을 제시해주었다.


좋아요!  
**Astro + Tailwind CSS**로  
`개발-블로그.html`과 같은 디자인의 마크다운 기반 블로그를 만드는 방법을  
**프로젝트 생성 → 디자인 적용 → 게시글 작성 → GitHub Pages 배포**  
순서로 안내드릴게요.

---

1. Astro 블로그 프로젝트 생성

```sh
npm create astro@latest
# 프로젝트명: blog (또는 원하는 이름)
# 템플릿: blog 선택 (또는 minimal 선택 후 직접 구조 추가)
cd blog
npm install
```

---

## 2. Tailwind CSS, Remixicon 등 스타일 세팅

```sh
npm install -D tailwindcss
npx tailwindcss init
```
- `astro.config.mjs`와 `tailwind.config.mjs` 연결
- `src/styles/global.css`에 Tailwind import
- Remixicon:  
  `<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/remixicon/4.6.0/remixicon.min.css">`  
  를 `src/layouts/Layout.astro`의 `<head>`에 추가

---

## 3. 폴더 구조 및 샘플 게시글

```
/src
  /components
    BlogCard.astro
    TagFilter.astro
    Sidebar.astro
    SearchBar.astro
  /content
    /blog
      2024-06-10-hello-world.md
      2024-06-11-react-patterns.md
  /pages
    index.astro
/public
  (이미지 등)
```

**샘플 마크다운 게시글**  
`src/content/blog/2024-06-10-hello-world.md`
```markdown
---
title: "모던 JavaScript 개발 환경 구축하기"
pubDate: "2024-12-15"
tags: ["JavaScript", "ES6+"]
category: "웹개발"
views: 1234
likes: 42
image: "/images/js-dev.jpg"
description: "최신 JavaScript 개발 환경을 구축하는 방법과 필수 도구들을 소개합니다. Webpack, Babel, ESLint 설정부터 개발 워크플로우 최적화까지..."
---
여기에 본문을 작성하세요!
```

## 4. 주요 컴포넌트 예시

### BlogCard.astro
```astro
---
const { title, pubDate, tags, views, likes, image, description } = Astro.props;
---
<article class="blog-card bg-white rounded-lg shadow-sm border border-gray-200 overflow-hidden">
  <img src={image} alt={title} class="w-full h-48 object-cover object-top" />
  <div class="p-6">
    <div class="flex items-center gap-2 mb-3">
      {tags.map(tag => <span class="px-2 py-1 bg-gray-100 text-xs rounded text-secondary">{tag}</span>)}
    </div>
    <h3 class="text-lg font-semibold mb-2 text-primary">{title}</h3>
    <p class="text-secondary text-sm mb-4 line-clamp-3">{description}</p>
    <div class="flex items-center justify-between text-xs text-secondary">
      <span>{pubDate}</span>
      <div class="flex items-center gap-3">
        <span class="flex items-center gap-1"><i class="ri-eye-line text-xs"></i>{views}</span>
        <span class="flex items-center gap-1"><i class="ri-heart-line text-xs"></i>{likes}</span>
      </div>
    </div>
  </div>
</article>
```

## 5. 메인 페이지(index.astro)에서 게시글 목록 렌더링

- Astro의 `getCollection`으로 md 파일 읽기
- BlogCard로 반복 렌더링
- 상단 검색, 태그 필터, 사이드바 등 추가

## 6. GitHub Pages 배포

1. `astro.config.mjs`에서 `output: 'static'` 설정
2. `npm run build`
3. GitHub Actions 워크플로우 추가  
   (Astro 공식문서의 [GitHub Pages 배포 가이드](https://docs.astro.build/ko/guides/deploy/github/) 참고)

# 프로젝트 구조 및 구현 내용 정리

## 프로젝트 개요

- **프로젝트명:** superdevsong.github.io
- **설명:** Astro, Tailwind CSS, MDX 기반의 블로그/정적 사이트 프로젝트
- **배포 주소:** [https://superdevsong.github.io](https://superdevsong.github.io)

---

## 사용 기술 스택

- **Astro**: 정적 사이트 생성기(Static Site Generator)
- **Tailwind CSS**: 유틸리티 퍼스트 CSS 프레임워크
- **MDX**: Markdown + JSX 지원
- **GitHub Actions**: CI/CD(자동 배포)
- **Node.js**: 패키지 관리 및 빌드

---

## 주요 구현/설정 내용

### 1. Astro 설정 (`astro.config.mjs`)

- 사이트 주소, Tailwind, MDX 통합 등 주요 설정이 포함되어 있습니다.
- 예시:
  ```js
  export default defineConfig({
    site: 'https://superdevsong.github.io',
    integrations: [
      tailwind(),
      mdx()
    ],
  });
  ```

### 2. Tailwind CSS

- `tailwind.config.js` 파일이 존재하며, Tailwind CSS가 통합되어 있습니다.
- Astro와 연동하여 유틸리티 CSS를 사용할 수 있습니다.

### 3. MDX 지원

- `@astrojs/mdx` 플러그인을 통해 Markdown과 JSX를 함께 사용할 수 있습니다.

### 4. GitHub Actions 배포

- `.github/workflows/deploy.yml` 파일을 통해 자동 배포가 설정되어 있습니다.
- 커밋/푸시 시 자동으로 빌드 및 배포가 이루어집니다.

### 5. 소스 코드 구조

- `src/components/`: 재사용 가능한 UI 컴포넌트
- `src/layouts/`: 페이지 레이아웃
- `src/pages/`: 라우팅되는 페이지
- `src/content/`: 블로그 포스트 등 컨텐츠
- `src/styles/`: 커스텀 스타일

### 6. 기타 설정

- `.gitignore`: 빌드 산출물, 환경변수, macOS 파일, IDE 설정 등 제외
- `.vscode/`: 개발 환경 통일을 위한 에디터 설정
- `tsconfig.json`: TypeScript 프로젝트 설정
